# 银行存贷业务建模

## 一、理解业务需求

```
	存款本质上来说是银行的负债，业务一般关心的是存款需要的成本以及存款的稳定性。因此在建模的时候就需要刻画出资金成本、客户结构以及资金流动性。
	贷款本质上就是是银行的资产，业务一般关心的是贷款带来的收益和风险。因此在建模的时候需要刻画利息收入、信用风险以及资产质量。
```

## 二、逻辑建模

### 2.1、确定业务过程

```
1、存款领域：
	存款交易：客户发生的一笔存款账户动账活动（存入、支取、结息）。
	存款账户余额快照：银行每日日终对存款账户状态的记录。
2、贷款领域：
	贷款发放：银行与客户签订合同并实际放款的关键事件。
	贷款还款计划执行：客户每偿还一期贷款（包括本金、利息、罚息）的事件。
	贷款合同状态快照：银行每日日终对贷款合同整体状态（余额、风险分类）的记录。
```

### 2.2、确定粒度

```
1、存款交易：单笔交易流水。每一行代表一笔不可再分的动账记录。
2、存款账户余额快照：每个账户每天一条记录。周期快照表
3、贷款发放：单笔贷款合同。每一行代表一笔独立的贷款。
4、贷款还款计划执行：单笔贷款合同的单期还款计划及其执行结果。这是理解现金流和风险的核心粒度。
5、贷款合同状态快照：每个贷款合同每天一条记录。
```

### 2.3、确定维度

```
1、公共维度：
	时间维度：日期、月份、季度、年份、是否节假日等。
	客户维度：客户编号、名称、类型（对公/个人）、风险等级、所属行业等。需要采用拉链表追踪历史变化。
	机构维度：支行、分行、业务部门等组织架构。
	员工维度：员工号、客户经理、柜员等。
	产品维度：存款/贷款产品名称、类型、基准利率、期限档位等。利率变更也需要使用拉链表。2、存款交易业务特定维度：
	交易渠道维度：柜面、网上银行、手机银行、ATM
	交易类型维度：现金存入、转账、利息结算、手续费代扣收。
3、贷款业务特定维度：
	还款方式维度：等额本息、等额本金、到期还本付息。
	五级分类维度：正常、关注、次级、可疑、损失。
	担保方式维度：信用、保证、抵押、质押。
	贷款用途：经营、消费、购房
```

### 2.4、确定事实

```
1、事务事实表（对应交易、发放等事件）：
	存款交易流水事实：交易金额、交易后余额、本次利息金额。
	贷款发放事实：合同金额、实际发放金额、执行年利率。
2、周期快照事实（对应每日余额、状态）：
	存款账户日余额快照事实：当前余额、日均余额、余额积数
	贷款合同状态事实：贷款余额、正常余额、逾期余额、应计利息。
3、累计快照事实（对应有明确工作流的过程，如还款计划）：
	贷款还款计划执行事实：应还本金、应还利息、实还本金、实还利息、实还罚息、逾期天数、剩余本金
```
